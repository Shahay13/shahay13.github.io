Lab 4

Developing an app with classes in Dart using Flutter.

View More

In Lab 4, "Android Studio: App Development - Classes," I explored the concept of classes in Dart. I learned how to define classes and their properties, implement constructors, and utilize inheritance and method overriding. Through practical examples, I understood how classes model real-world objects, promote code reuse, and enhance modularity. The lab also highlighted the differences between Dart's classes and those in other programming languages, emphasizing the advantages of using classes for structured and maintainable code in mobile development.

Code Snippets (Dart)


// Create a class called Car.
class Car {
  // Set properties for the class.
  String make;   // Make of the car (for example, Nissan).
  String model;  // Model of the car (for example, Corolla).
  int year;      // Year of the car (for example, 2015).

  // Create a constructor to set the properties.
  Car(this.make, this.model, this.year);

  // Function to show car information.
  void showInfo() {
    // Print the make, model, and year of the car.
    print('Car: $make $model, Year: $year');
  }
}

// Main method where the program runs begins.
void main() {
  // Make the car class an object.
  var myCar = Car('Nissan', 'Corolla', 2015);
  // Call the show info function on the car object.
  myCar.showInfo();
}

Implementing Material Components to build a task list interface.

In Lab 5, "Introduction to Flutter," I learned how to use Material Components in Flutter to create a user-friendly and visually appealing task management application. I implemented various Material Buttons (ElevatedButton, OutlinedButton, and TextButton) and Material Cards to display task information effectively. Through this exercise, I enhanced my understanding of Flutter's design components and their application in real-world scenarios, improving my skills in mobile app development.

Code Snippets (Dart)


import 'package:flutter/material.dart';

// The opening code for the app.
void main() {
  // Executes the application by making an example of my app.
  runApp(const MyApp());
}

// My app class that extends stateless widget, which means it is fixed.
class MyApp extends StatelessWidget {
  // The constructor with a key to handle the widget state.
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    // Makes the widget tree for the app.
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(
          title: Text('Material Buttons Example'),
        ),
        body: Center(
          child: Column(
            mainAxisAlignment: MainAxisAlignment.center,
            children: <Widget>[
              // The elevated button widget is used for main activities.
              ElevatedButton(
                onPressed: () {
                  // Prints the text when the button is clicked.
                  print('Elevated button clicked.');
                },
                // The text is shown on the button.
                child: Text('Elevated Button'),
              ),
              // The outlined button widget is used for subordinate activities.
              OutlinedButton(
                onPressed: () {
                  // Prints the text when the button is clicked.
                  print('Outlined button clicked.');
                },
                // The text is shown on the button.
                child: Text('Outlined Button'),
              ),
              // The text button widget is used for less significant activities.
              TextButton(
                onPressed: () {
                  // Prints the text when the button is clicked.
                  print('Text button clicked.');
                },
                // The text is shown on the button.
                child: Text('Text Button'),
              ),
            ],
          ),
        ),
      ),
    );
  }
}

Classwork 3

Creating a functional mobile app using Flutter.

View More

In classwork 3, “Flutter: App Development,” moving from Android Studio to Flutter was both hard and fun. Using XML for layout design in Android Studio might be useful, but it can also be tedious and complicated for advanced interfaces. The challenges I faced were changing to the Dart programming language and Flutter's widget architecture and learning the widget tree structure. The advantages are that Flutter helps cross-platform development, hot reloading for fast changes, and a big collection of already built widgets, and Android Studio gives full design tools, improved interaction with specific Android tools, and strong debugging features. The disadvantages are that Flutter needs understanding new structures and language, and Android Studio's user interface build step is more complicated and takes time. While Android Studio's extensive tools make it suitable for making specific Android applications, Flutter's multi-platform support and fast creation step make it an outstanding option for programmers who want to rapidly make apps for many platforms.
